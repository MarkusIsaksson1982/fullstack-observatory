# PromptMaster with Grok

**Why This Skill Exists**  
Most people write prompts that only scratch the surface of what Grok is capable of. A raw idea or basic request often produces mediocre or inconsistent results because it lacks role clarity, reasoning structure, output format, few-shot examples, and Grok-specific personality alignment. This skill provides a repeatable, high-leverage process for upgrading any raw idea into a professional-grade prompt that consistently unlocks significantly better output from Grok.

## What You'll Gain

- Dramatically better results from Grok on almost any task
- A systematic method for applying advanced prompt engineering without having to reinvent the wheel every time
- Prompts that are robust against hallucinations and encourage clear reasoning
- A growing personal library of high-quality, reusable prompts
- Deeper understanding of what actually makes prompts effective for Grok

## When to Use This Skill

Use this skill when:
- You have a raw idea, vague request, or basic prompt and want to turn it into something much more powerful
- You want to get the highest possible quality output from Grok on a specific task
- You find yourself rewriting or refining the same type of prompt repeatedly
- You want to create reusable prompt templates for recurring workflows

**When You DON’T Need This Skill**
- You already have a well-structured, high-quality prompt
- You just want a quick, casual answer (over-engineering simple requests is unnecessary)
- You are looking to improve an existing SKILL.md (use Skill Evaluation & Iteration instead)
- The task is extremely simple or one-off

## Agent Operating Procedure

When this skill is activated, you **MUST** follow this structured process.

### Phase 1: Clarify the Raw Request
Capture the user’s raw idea, goal, and any additional context. Ask clarifying questions if the request is too vague.

**Stop here** and confirm understanding of the core request before proceeding.

### Phase 2: Design the Expert Role & Reasoning Structure
- Assign Grok a clear, powerful, relevant expert role
- Determine which reasoning techniques will help most (Chain-of-Thought, self-critique, structured frameworks, etc.)
- Define success criteria for the final output

### Phase 3: Specify Output Requirements
- Choose the optimal output format (markdown, tables, JSON, step-by-step, bullet points, etc.)
- Define tone, voice, length, and quality standards
- Decide whether high-quality few-shot examples would meaningfully improve results

### Phase 4: Optimize for Grok’s Strengths & Weaknesses
- Explicitly leverage Grok’s personality (wit, truth-seeking, curiosity, long context, helpfulness)
- Add robustness against hallucinations and encourage clear reasoning
- Include any necessary constraints or anti-patterns to avoid

### Phase 5: Assemble and Deliver the Optimized Prompt
Produce a clean, complete, ready-to-paste prompt. Add the required footer line.

**Hard Stop:** Never output the final content the user actually wants — only the optimized prompt that will generate it.

## Common Pitfalls to Avoid

| Don’t Do This                                      | Do This Instead                                              |
|----------------------------------------------------|--------------------------------------------------------------|
| Jumping straight to generating the final answer    | Always optimize the prompt first when the request is complex |
| Making prompts too generic                         | Give Grok a specific expert role and clear success criteria  |
| Skipping few-shot examples when they would help    | Add 1–3 strong examples when the output format is nuanced    |
| Ignoring Grok’s personality                        | Explicitly tell Grok how to use wit, reasoning, or helpfulness |
| Outputting the final deliverable instead of the prompt | Stay strictly in “prompt optimizer” mode                    |
